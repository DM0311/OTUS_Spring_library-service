# Библиотечный сервис (Library Service)

## Описание
RESTful-сервис для управления библиотекой. Позволяет пользователям искать книги, бронировать их, оставлять отзывы, получать уведомления. Администратор может управлять каталогом, пользователями и просматривать аудит.

## Технологический стек
- Java 17
- Spring Boot 3.4.4
- Spring Data JPA (Hibernate)
- PostgreSQL 17
- Flyway (миграции)
- Spring Security + JWT
- MapStruct 
- Lombok
- Swagger/OpenAPI

## Структура проекта
* controllers – REST-контроллеры
* dto – объекты передачи данных для запросов/ответов
* events - события для уведомлений и аудита
* exception - исключения и глобальный обработчик
* listeners - слушатели событий аудита и уведомлений
* mappers - Mapstruct-мапперы
* model/entity – сущности
* repositories – JPA-репозитории
* service – бизнес-логика

## Функциональность
### Для всех пользователей (без аутентификации)
- Регистрация – создание учётной записи с ролью USER.
- Просмотр книг – получение списка всех книг, поиск по названию, автору, жанру, году.
- Просмотр деталей книги – информация о книге, авторах, жанрах, рейтинге, доступных экземплярах.

### Для авторизованных пользователей (после внедрения JWT)
- Бронирование книги – создание аренды, если есть доступные экземпляры.
- Продление бронирования – увеличение срока аренды (максимум 1 раз).
- Возврат книги – закрытие бронирования и увеличение доступных копий.
- Отмена бронирования – отмена активного бронирования.
- История бронирований – просмотр своих аренд с фильтром по статусу.
- Отзывы на книги – добавление, редактирование и удаление комментариев с рейтингом (1–5).
- Уведомления – просмотр системных уведомлений (о просрочке, начислении штрафов, блокировке, напоминаниях) и отметка о прочтении.

### Для администратора
- Управление книгами – создание, редактирование, удаление книг (связи с авторами и жанрами).
- Управление авторами и жанрами – CRUD-операции.
- Управление пользователями – просмотр списка, блокировка/разблокировка, назначение ролей.
- Управление бронированиями – просмотр всех аренд, принудительное закрытие/отмена.
- Аудит действий – просмотр журнала событий (кто, когда, что делал).
- Статистика (в разработке) – топ популярных книг, активные читатели, количество просрочек.

### Автоматические фоновые задачи (Scheduler)
- Проверка просроченных бронирований – ежедневное начисление штрафных баллов за каждый день просрочки.
- Блокировка пользователей – автоматическая блокировка при превышении 10 штрафных баллов (сроком на 30 дней).
- Разблокировка – автоматическое снятие блокировки по истечении срока.
- Напоминания – отправка уведомлений за 3 дня до даты возврата.

### Событийная система (аудит и уведомления)
- Аудит – все изменяющие действия (создание, обновление, удаление, бронирование и т.д.) логируются через события.
- Уведомления – генерируются автоматически при просрочке, начислении штрафа, блокировке/разблокировке и доступны пользователю в личном кабинете.

   ```bash

     -e POSTGRES_PASSWORD=postgres \
     -e POSTGRES_USER=postgres \
     -e POSTGRES_DB=library_db \
     -p 5432:5432 \
     postgres:17
Запустить приложение:

bash
./mvnw spring-boot:run
